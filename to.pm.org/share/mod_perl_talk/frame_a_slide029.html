"Kits" for mod_perl

  • There exist many application / template toolkits for mod_perl
    • Remember, it is these that you should compare against PHP, Cold Fusion, ASP/VB, etc.
  • They exist in order to help developers create bigger and better applications easier than just writing dumb CGI code
  • Some of them are exclusively mod_perl oriented (e.g. PageKit)
  • Some take no intrinsic advantage of advantage mod_perl (e.g. HTML::Template)
  • Some are "ambidextrous" between mod_perl, CGI, other accelerators & frameworks (e.g. AxKit, EmbPerl, Mason)
