mod_perl benchmarks, debugging, status checking

  • Joshua Chamas' benchmarking project - http://www.chamas.com/bench/
    • Architecture matters (e.g. XML/XSLT engines are dog-slow)
    • Implementation matters (e.g. Apache::ASP vs. HTML::Embperl)
    • Operating system, web server, language matter less
  • Debugging is much like Perl debugging
    • print STDERR Data::Dumper->Dump([...],[...]);
    • use CGI::Carp qw(fatalsToBrowser); # works well for Apache::Registry scripts
    • debugging via ptkdb didn't work the last time I tried (2 years ago) (this does work for CGI scripts!)
  • Status checks abound
    • http://localhost/server-status - intrinsic to Apache
    • http://localhost/perl-status - per Apache::Status, comes with mod_perl
    • http://localhost/extended-system-monitor - per Apache::VMonitor, more of an Apache than mod_perl check
