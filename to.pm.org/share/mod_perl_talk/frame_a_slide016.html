mod_perl handlers

  • Essentially, for each step in the transaction look, there is a "hook" in the mod_perl API that ties into the Apache API, such that you can register a Perl module as the "handler" for that step in the loop!
  • You perform this "registration" in the httpd.conf file
  • Different handlers are responsible for creating "responses" to the Apache API that make sense at the particular stage at which the handler is registered
    • E.g. Reponse phase -> PerlHandler directive
    • URI Translation phase -> PerlTransHandler
  • (The names PerlHandler and PerlTransHandler are configuration directives within httpd.conf... more later)
  • The response codes are found in the Apache::Constants package (comes with mod_perl package)
    • A PerlHandler returns "OK" if it successfully "does something"
    • It returns "DONE" if it does something and that should abort the rest of the loop processing
    • It returns "DECLINED" if it doesn't want to handle the request
    • It may return any one of a number of different error codes (SERVER_ERROR, UNAUTHORIZED, REDIRECT, NOT_FOUND, etc.)
