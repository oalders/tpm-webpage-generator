What's wrong with CGI?

  • Plenty
  • It's slow -- spawning new processes with each http hit
    • (Especially slow when you use an interpreted language like Perl, needing to start up a heavyweight interpreter first)
  • It's resource intensive -- again, spawning new processes
  • It's subseptible to resource-usage "spikes" -- again, spawning processes
  • It isn't "close to the server", thus preventing it from doing certain types of things (more on this later)
  • It doesn't actually help you create web applications -- it defines no techniques, it provides no libraries, etc.
