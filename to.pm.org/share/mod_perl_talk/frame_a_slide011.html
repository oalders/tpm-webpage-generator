Why program Apache API handlers in Perl? Why not use C? (Expanded Answer)

  • If you want to ask this question, you're in the wrong lecture
  • This question mainly falls into 2 parts:
    • Why program the Apache API at all?
    • Why use C over Perl?
  • Why program the Apachi API? To fix all the problems that CGI had, and to gain new abilities (already discussed)
  • Why program it in Perl rather than C?
    • So that you don't have to recompile your Apache httpd binary every time you want to change a printf
    • Because Perl is a "nicer" language then C in general
    • Because so much of web programming involves manipulating HTTP and HTML, both of which are text, and Perl kicks butt at text manipulation
    • Because Perl is great for access to databases, too, via the DBI
    • Because of CPAN in general
  • Basically, the same reasons you'd want to program in Perl in any possible situation
