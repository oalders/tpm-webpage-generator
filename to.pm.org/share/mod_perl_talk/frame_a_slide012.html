Benefits of mod_perl over CGI

  • Essentially, its good points are the opposites of all of the criticism of CGI
  • It's fast -- it is loaded when Apache is loaded, because it is part of Apache! You only need to "load" the Perl interpreter once.
  • It's fast -- it caches compiled Perl op-codes for any given piece of Perl you run through it
  • It's fast -- since it's always around, it can "hold onto" database connections, saving time
  • It's more resource-predictable (but still it can have some problems, specifically memory usage due to caching)
  • While not itself particularly easy to program, there are many "web toolkits" that build on it that make "CGI-ish" programming much easier
