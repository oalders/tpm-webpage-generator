Apache::PerlRun

  • This is a wrapper for quick-and-dirty Perl/CGI scripts that "aren't well behaved"
    • don't use strict
    • are careless with closures
    • are prone to memory leaks
    • aren't careful with filehandles (e.g. closing them afterwards)
  • If you don't consciously program Perl carefully, then this is for you
  • It "speeds up" the Perl/CGI program because you don't have to load the Perl interpreter and spawn a new process
  • However, no other speed-up that is possible through mod_Perl is implemented
    • It loads your code once, runs it, and promptly drops it like the radioactive waste it is
  • I wanted to create an Apache::PerlRun example, but I just can't code that badly
